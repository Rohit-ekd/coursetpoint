SQL Set Operation

Set Operations in SQL are used to combine or manipulate the result sets of multiple SELECT queries.

They allow you to perform operations similar to those in set theory, such as union, intersection, and difference, on the data retrieved from different tables or queries.

Set Operations provide powerful tools for managing and manipulating data, enabling you to analyse and combine information in various ways.

There are four primary set operations in SQL:

  • UNION
  • INTERSECT
  • EXCEPT (or MINUS)
  • UNION ALL
  1. UNION: The UNION operator combines the result sets of two or more SELECT queries into a single result set.

    2. It removes duplicates by default, meaning that if there are identical rows in the result sets, only one instance of each row will appear in the final result.

    Example: Assume we have two tables: Customers and Suppliers.

    Customer Table:

    CustomerID CustomerName
    1 Alice
    2 Bob

    Suppliers Table:

    SuppliersID SuppliersName
    101 SupplierA
    102 SupplierB

    UNION Query:

    SELECT CustomerName FROM Customers UNION SELECT SuppliersName FROM Suppliers;

    Result:

    CustomerName
    Alice
    Bob
    SupplierA
    SupplierB
  2. INTERSECT: The INTERSECT operator returns the common rows that exist in the result sets of two or more SELECT queries.

    3. It only returns distinct rows that appear in all result sets.

    Example: Using the same tables as before.

    SELCT CustomerName FROM Customers INTERSECT SELECT SupplierName FROM Suppliers;

    Result:

    CustomerName
  3. EXCEPT: The EXCEPT operator (also known as MINUS in some databases) returns the distinct rows that are present in the result set of the first SELECT query but not in the result set of the second SELECT query.

    4. Example: Using the same tables as before.

    SELECT CustomersName FROM Customers ECEPT SELECT SupplierName FROM Suppliers;

    Result:

    CustomerName
    Alice
    Bob

    In this example, the names "Alice" and "Bob" are customers but not suppliers, so they appear in the result

  4. UNION ALL: The UNION ALL operator performs the same function as the UNION operator but does not remove duplicates from the result set. It simply concatenates all rows from the different result sets.

    Example: Using the same tables as before.

    SELECT CustomerName FROM Customers UNION ALL SELECT SupplierName FROM Supplier;

    Result:

    CustomerName
    Alice
    Bob
    SupplierA
    SupplierB
