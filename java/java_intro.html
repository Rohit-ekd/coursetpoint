Features of Java
  1. Simple: Java was designed to be easy for the programmers to learn and use effectively. If you already understand the basic concept of object oriented programming, learing java will even easier.
  2. Object Oriented: In starting we can say java is pure OOP, Everything in java is an Object. It simplifies software developement and maintained. The Object model in java is simple and easy to extend, while primitive type, such as integers, are kept as high performance nonobject.

  3. Platform Independent: Java is platform Independent means other languages like c/c++ are machine and platform dependent where java is platform dependent that means code of java run on different platform like windows ,mac and linux gives the same output.
  4. Secured: Java is best known for his Security with java we can developed virus free software such as:

    • Due to Strong Memory Management.
    • Java handles exceptions easy compare to other Programming.
    • Java programs making less prone to crashes and errors.
    • Automatic garbage collection facility provide by java.
  6. Portable: Java is portable because it facilitates you to carry the java byte code to any platform. It does not required any other implementation.

Creating the Program

We can create a prgram using Notepad text editor. Assume we have entered the following program:

                            
public class First {
    public static void main(String[] args) {
        System.out.println("Hello, courseTpoint!");
        System.Out.println("Welcome to the world of java.")

    }
}

We must save this program in a file called First.java ensuring that the filename contains the class name properly. This file is called the source file.

Compiling the Program

To commpile the program, we must run the java Compiler javac, with the name of the source file on the command line as shown below:

javac First.java

If everything is Ok, the javac compiler creates a file called First.class containing the bytecodes of the program. < classname >.class

Running the Program

We need the to use the java interpreter to run a stand-alone program. at the command Prompt type

java First

JVM(Java Virtual Machine)

It provides the runtime environment in which Java bytecode can be executed. Here's an overview of the JVM in Java.

It is a program that interprets the intermediate java byte code and generate output. It is because of bytecode and JVM concept that makes java highly portable.

It's responsible for interpreting and executing Java bytecode, which is the intermediate representation of Java source code compiled from .java files.

  • Class Loader: Loads class files into memory.
    • Loading
    • Linking
    • Initialing
  • Bytecode Verifier: Checks the bytecode for security and integrity.
  • Platform Independence:Java bytecode is platform-neutral, meaning it can be executed on any device that has a compatible JVM installed, regardless of the underlying hardware or operating system.
  • Memory Management:
    • When you compile a Java source file (`.java`), the Java compiler (`javac`) generates bytecode (`.class` files).
    • The JVM loads these bytecode files and executes them.
    • The JVM dynamically translates bytecode into native machine code using the JIT(just-in-time) compiler, optimizing performance.
JRE(Java Runtime Environment)

JVM is a part of JRE. JRE is facilitates the execution of programs developed in Java. It is used to provide runtime Environment that implementating in JVM.

The JRE is a software package that includes the Java Virtual Machine (JVM), libraries, and other components necessary to run Java applications.
It does not contain development tools such as compilers and debuggers; those are provided by the Java Development Kit (JDK).

Components of JRE

  • JVM(Java Virtual Machine)
  • Run time Libraries
  • User Interface Toolkits
  • Deployment technologies
    • Java plug-in
    • Java Web start
JDK(Java Developement Kit)

JDK is provide a collection of tools that are used to running java programs and developed Java-based software applications and applets.
